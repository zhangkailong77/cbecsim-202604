最后更新：2026-03-19

- 当前系统主链路已覆盖：`Step 01 开局 -> Step 02 选品采购 -> Step 03 物流清关 -> Step 04 入仓 -> Step 05 店铺运营（Shopee）`。

- 已实现物流“自动推进 + 手动加速并存”：
  - 自动推进按游戏时间 tick 执行（在订单查询链路自动补推进）。
  - 手动推进按钮保留，作为即时加速下一节点。
- 订单页补充运营效率交互：
  - 待出货页支持勾选、全选本页与批量发货。
  - 订单列表支持分页浏览（URL 保留 `page` 参数，含页码省略号样式）。
  - 后端已新增常驻自动订单tick worker：按游戏时钟持续补跑订单（默认每 15 秒循环、每次每个 run 最多补 2 个游戏小时），无需手动刷新页面。
  - 订单列表保留轻量兜底补偿（单次最多补 1 tick），避免首次加载大批量补算导致接口变慢。
  - 订单自动生成节奏已调整：从“每 1 游戏小时一个 tick”改为“每 8 游戏小时一个 tick”（约每 10 分钟真实时间一轮，worker 与订单页兜底补偿保持一致）。
  - 物流自动推进已调整为“按运输游戏天数里程碑推进”，不再按固定每小时推进节点：发货时会读取 Step03 最新线路（经济/标准/快速）并结合仓到买家距离计算预计运输天数，达到预计签收时间后才自动进入已签收。
  - 已修复“线路文案与 ETA 来源不一致”问题：发货时将真实履约线路（`delivery_line_key/label`）固化到订单，后续运输中列表与物流详情统一读取订单级线路；历史订单无该字段时会按 ETA 反推线路作为兜底展示。
  - 订单列表（运输中）与物流详情页已补充时效解释展示：配送线路、线路时效区间、预计/已运输/剩余天数，便于玩家理解 ETA 来源。
  - 已修复订单列表偶发 500：结算写入在并发推进下可能触发 `shopee_order_settlements.order_id` 唯一键冲突，现改为并发安全幂等 upsert（savepoint + flush，冲突时回退为 update）。
- 商品列表显示优化：
  - 已新增变体销量字段 `shopee_listing_variants.sales_count`（模型、增量补字段、字段注释、接口返回）。
  - 下单模拟会同步累加对应变体销量，商品列表支持按变体显示“销量 + 库存”真实值。
- 商品编辑语义修正：
  - 编辑已上架商品时，“更新”按钮改为仅更新商品信息（保持原上架状态，不再按“重新发布”语义处理）。
  - 编辑页底部操作文案统一中文：`取消 / 下架 / 更新`。
  - 编辑模式页面文案区分已完成：从“编辑”进入时，顶部面包屑末级与页面标题显示为`产品详情`；从“添加新商品”进入时仍保持`添加新商品`。
  - 已修复编辑更新导致“变体图片/变体销量丢失”问题：编辑发布时按 `source_variant_id -> SKU -> 变体选项` 关联旧变体，默认保留历史 `image_url` 与 `sales_count`，仅在重新上传变体图时覆盖图片。
  - 已修复编辑更新导致“变体 ID 变化”问题：编辑发布改为“就地更新已存在变体”，不再先清空重建；仅对确实被删除的变体做删除、对新增变体做新增，因此仅改库存等字段时变体 ID 保持不变。

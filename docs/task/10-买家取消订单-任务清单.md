# 10-买家取消订单-任务清单

> 范围说明：本清单仅实现“超时未发货 -> 买家取消”能力，不包含退货退款。
> 阈值口径：以游戏内对局时间为准，超时满 `48` 游戏小时后才开始计算取消概率。

## 1. P0-订单取消字段与注释补齐（先做）
- 状态：`已完成`
- 扩展 `shopee_orders`：
  - `cancelled_at`：取消时间
  - `cancel_reason`：取消原因（如 `seller_not_ship_in_time`）
  - `cancel_source`：取消来源（`buyer_auto` / `manual_debug`）
- 维护 `db.py` 增量补字段逻辑。
- 补齐 table comment / column comment。
- 验收标准：模型、表结构、注释一致，历史库可自动补齐。

## 2. P0.5-物流轨迹取消事件语义扩展
- 状态：`已完成`
- 在 `shopee_order_logistics_events` 事件语义中新增：
  - `cancelled_by_buyer`
- 事件标题建议：`买家取消订单`
- 事件描述建议：`卖家超时未发货，买家取消订单`
- 验收标准：订单取消后可在轨迹中看到取消事件。

## 3. P1-自动取消服务（游戏时间驱动）
- 状态：`已完成`
- 在 Shopee 履约路由/服务中新增内部函数：
  - `auto_cancel_overdue_orders_by_tick(run_id, user_id, current_tick)`
- 规则：
  - 仅 `type_bucket=toship` 参与判定
  - `overdue_hours = floor((current_tick - ship_by_at)/1h)`
  - `overdue_hours < 48`：取消概率为 0
  - `overdue_hours >= 48`：`cancel_prob = min(0.90, 0.25 + (overdue_hours - 48) * 0.08)`
- 命中后动作：
  - 订单状态 -> `cancelled`
  - 写入 `cancelled_at/cancel_reason/cancel_source`
  - 写入轨迹 `cancelled_by_buyer`
- 验收标准：游戏时间推进后，超时订单会按概率自动转为取消。

## 4. P1.5-手动取消调试接口
- 状态：`已完成`
- 新增接口：`POST /shopee/runs/{run_id}/orders/{order_id}/cancel`
- 校验：
  - 仅 `toship` 可取消
  - 非 `toship` 返回拒绝
- 默认参数：
  - `reason=seller_not_ship_in_time`
  - `cancel_source=manual_debug`
- 验收标准：可手动触发取消，状态与轨迹正确落库。

# 09-履约发货物流结算-任务清单

> 状态（2026-03-18）：已完成 P0 ~ P4（手动物流推进版），P5 测试补强与 P6 文档同步持续完善中。

## 1. P0-地理数据补齐（先做）
- 买家池新增字段：`city_code`、`lat`、`lng`。
- 字段作用：
  - `city_code`：统一城市映射键，便于按城市配置规则。
  - `lat/lng`：直接参与仓点-买家距离计算。
- 仓库点位复用已有 `warehouse_landmarks` 坐标。
- 验收标准：50 个买家均有可用地理信息（坐标或可映射）。

## 2. P0.5-订单与结算字段扩展
- 扩展 `shopee_orders`：
  - `tracking_no`：快递追踪号。
  - `waybill_no`：面单号。
  - `ship_by_at`：最晚发货时间（逾期判定）。
  - `shipped_at`：实际发货时间。
  - `delivered_at`：签收时间。
  - `eta_start_at` / `eta_end_at`：预计送达区间。
  - `distance_km`：仓到买家距离。
- 新增 `shopee_order_logistics_events`（物流轨迹表）。
- 新增 `shopee_order_settlements`（结算表）。
- 验收标准：表结构、ORM、表注释/字段注释齐全。

## 3. P1-运费与时效计算服务
- 新增服务文件：`backend/apps/api-gateway/app/services/shopee_fulfillment.py`。
- 实现函数：
  - `haversine_km(warehouse_latlng, buyer_latlng)`
  - `calc_shipping_cost(distance_km, shipping_channel)`
  - `calc_eta(distance_km, shipping_channel)`
  - `gen_tracking_no()` / `gen_waybill_no()`
  - `calc_settlement(...)`
- 验收标准：函数可独立测试，输入固定时输出稳定。

## 4. P2-安排发货接口（核心）
- 新增接口：`POST /shopee/runs/{run_id}/orders/{order_id}/ship`。
- 处理逻辑：
  - 仅 `toship` 状态允许发货。
  - 物流渠道必须属于商品上架时启用渠道。
  - 生成 `tracking_no`、`waybill_no`。
  - 计算 `distance_km`、`shipping_cost`、`eta_start_at`、`eta_end_at`。
  - 写入首条轨迹节点 `label_created`。
  - 订单状态切换为 `shipping`。
- 验收标准：待出货订单可进入运输中，单号与 ETA 可见。

## 5. P2.5-物流详情与轨迹推进
- 新增接口：`GET /shopee/runs/{run_id}/orders/{order_id}/logistics`。
- 新增调试接口：`POST /shopee/runs/{run_id}/orders/{order_id}/logistics/progress`（先手动推进）。
- 节点流转：
  - `label_created` -> `picked_up` -> `in_transit` -> `out_for_delivery` -> `delivered`
- 验收标准：详情页可查看完整时间线，节点流转合法。

## 6. P3-签收结算
- 在 `delivered` 节点触发：
  - 订单状态 -> `completed`
  - 写入 `shopee_order_settlements`
- 新增接口：`GET /shopee/runs/{run_id}/orders/{order_id}/settlement`
- 结算公式：
  - `net_income = buyer_payment - platform_commission - payment_fee - shipping_cost + shipping_subsidy`
- 验收标准：签收后可查看可核对的结算明细。

## 7. P4-前端对齐 Shopee 履约流程
- 页面：`frontend/src/modules/shopee/views/MyOrdersView.tsx`
- 任务：
  - 待出货状态增加“安排发货”按钮并接接口。
  - 运输中显示 tracking_no、ETA 区间。
  - 操作列三项（查看物流详情/查看详情/打印面单）接真实行为。
  - 结算详情入口对接 settlement 接口。
- 验收标准：玩家端可完整完成发货 -> 查轨迹 -> 签收 -> 看结算。

## 8. P5-测试与回归
- 后端测试新增：

  - 发货后单号、距离、运费、ETA 正确写入。
  - 轨迹推进后状态合法。
  - `delivered` 后结算记录存在且净入账正确。

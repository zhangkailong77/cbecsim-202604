# 11-卖家回款与余额流水-任务清单

> 范围说明：本清单仅实现“已完成订单回款 + 余额流水展示”能力，不包含提现、贷款、Fast Pay、退款冲销。
> 时间口径：全部以游戏内对局时间为准（`current_tick`）。

## 1. P0-财务流水表与注释补齐（先做）
- 状态：`已完成`
- 新增表：`shopee_finance_ledger_entries`（建议命名）
- 字段建议：
  - `run_id`、`user_id`、`order_id`
  - `entry_type`（`income_from_order` 等）
  - `direction`（`in/out`）
  - `amount`
  - `balance_after`
  - `status`
  - `remark`
  - `credited_at`、`created_at`
- 约束建议：
  - 唯一键：`(order_id, entry_type)`（防重复订单回款）
- 维护 `db.py` 的增量补表/补字段逻辑。
- 补齐 table comment / column comment。
- 验收标准：模型、表结构、注释一致，历史库可自动补齐。

## 2. P1-回款入账服务（幂等 + 事务）
- 状态：`已完成`
- 新增服务函数（建议）：
  - `credit_order_income_if_needed(run_id, user_id, order_id, current_tick)`
- 规则：
  - 仅 `completed` 且非 `cancelled` 订单可回款
  - 仅存在有效结算记录（`settled`）时可入账
  - 同订单仅入账一次（幂等）
- 写入内容：
  - 生成 `income_from_order` 流水
  - 记录 `balance_after` 快照
- 验收标准：重复触发不会重复加余额，并发场景不重复写入。

## 3. P1.5-回款触发点接入
- 状态：`已完成`
- 在订单从 `shipping -> completed` 的链路接入自动回款调用。
- 在订单查询兜底自动推进路径中，确保“到达 completed 后可补入账”。
- 验收标准：订单签收完成后能自动出现回款流水。

## 4. P2-财务接口（后端）
- 状态：`已完成（MVP）`
- 新增接口：
  - `GET /shopee/runs/{run_id}/finance/overview`
  - `GET /shopee/runs/{run_id}/finance/transactions`
  - `GET /shopee/runs/{run_id}/finance/income`
- 能力要求：
  - 支持分页
  - 支持基础筛选（方向/类型/时间范围）
  - 返回字段中文语义清晰（前端直接展示）
- 验收标准：接口可支撑“我的余额/我的收入”页面完整渲染。

## 5. P3-前端页面打通（Shopee 财务区）
- 状态：`已完成（MVP）`
- 菜单与路由：
  - `我的余额` -> `my-balance`
  - `我的收入` -> `my-income`
- 页面：
  - `MyBalanceView`：余额总览 + 流水筛选 + 交易列表
  - `MyIncomeView`：收入摘要 + 订单回款列表
- 交互要求：
  - 分页、空态、加载态
  - 与现有 Shopee 风格一致，保留返回工作台入口
- 验收标准：玩家可完整查看余额与回款记录。

## 6. P4-测试与回归
- 状态：`进行中`
- 后端测试新增：
  - completed 订单仅回款一次
  - cancelled 订单不回款
  - 并发/重复调用不重复入账
  - 分页与筛选结果正确
- 前端联调：
  - 余额与流水金额和后端一致
  - 收入列表与订单完成记录可对齐
- 验收标准：关键链路稳定，无重复入账与明显回归。

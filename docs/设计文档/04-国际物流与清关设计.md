# CBEC SIM 阶段设计文档（第四步）

## 阶段进度
- 第一步（已完成）：注册/登录
- 第二步（已完成）：开局准备
- 第三步（已完成）：选品与采购
- 第四步（当前）：国际物流与清关

## 第四步目标
- 玩家针对“待发运采购单”选择马来线路货代方案。
- 完成报关报检与清关策略选择。
- 计算并扣减本次物流与清关费用。
- 生成物流单，进入“在途/清关”过程，供后续海外仓入仓使用。

## 本阶段定位
- 这是采购后的履约衔接环节，不是店铺运营。
- 当前只支持目标市场 `MY`（马来）。
- 玩家可配置线路与清关策略；流程推进主要由系统时间驱动。

## 页面模块（Step 03）
1. A 区：待发运采购单
- 显示本局可发运采购订单（订单号、商品数、件数、货值、时间）。
- 支持勾选一个或多个订单发运。

2. B 区：货代线路选择（马来）
- 经济线（马来）：低费用、时效较慢、稳定性中。
- 标准线（马来）：中费用、中等时效、稳定性高。
- 快速线（马来）：高费用、时效最快、稳定性偏高。

3. C 区：清关策略
- 标准清关：额外费用低，清关天数较长。
- 加急清关：额外费用高，清关天数较短。

4. D 区：费用与时效预估
- 选中订单货值
- 物流费（按线路规则）
- 清关费（按策略规则）
- 本次总费用
- 扣款后余额
- 预计总时效（运输 + 清关）

5. E 区：物流清关进度
- 显示阶段：`已下单 -> 运输中 -> 清关中 -> 清关完成`
- 展示当前状态、预计完成时间、物流单号。

## 费用与时效规则（V1）
1. 费用组成
- 物流费：`货值 * 线路费率 + 线路固定费`
- 清关费：按策略固定费用或费率计算
- 本次总费用：`物流费 + 清关费`

2. 时效组成
- 运输天数：由线路决定
- 清关天数：由清关策略决定
- 总时效：`运输天数 + 清关天数`

3. 扣款规则
- 确认发运时一次性扣减 `本次总费用`。
- 资金不足时禁止发运并提示。

## 时间系统与推进规则
1. 全局映射
- 一局真实时长映射为 7 天（对应游戏内 365 天）。
- 页面展示统一倒计时（精确到秒，实时刷新）。

2. 阶段倒计时
- Step 03 使用“阶段截止时间 - 当前时间”的秒级倒计时。
- 倒计时到期前，物流与清关状态按系统时间自动推进。

3. 手动与自动边界
- 玩家负责“选择与确认发运”。
- 发运后的进度（运输/清关完成）由系统按时间自动推进。

## 数据模型（建议）
1. 物流单主表 `logistics_shipments`
- `id`
- `run_id`
- `user_id`
- `shipment_no`
- `market`（MY）
- `forwarder_plan`（economy|standard|fast）
- `customs_plan`（standard|express）
- `cargo_value`
- `logistics_fee`
- `customs_fee`
- `total_fee`
- `status`（created|in_transit|customs|completed）
- `created_at`
- `eta_at`
- `completed_at`

2. 物流单关联表 `logistics_shipment_orders`
- `id`
- `shipment_id`
- `procurement_order_id`
- `order_value`
- `order_quantity`

3. 游戏局资金字段（已有）
- `game_runs.cash_balance`：确认发运后扣减。

## 接口设计（建议）
1. `GET /game/runs/{run_id}/logistics/candidates`
- 返回可发运采购订单列表。

2. `POST /game/runs/{run_id}/logistics/estimate`
- 入参：`order_ids, forwarder_plan, customs_plan`
- 返回：费用预估、预计时效、扣款后余额。

3. `POST /game/runs/{run_id}/logistics/shipments`
- 入参：`order_ids, forwarder_plan, customs_plan`
- 服务端：资金校验 -> 扣款 -> 落物流单 -> 启动状态推进。
- 出参：`shipment_id, shipment_no, total_fee, cash_balance_after`

4. `GET /game/runs/{run_id}/logistics/shipments/latest`
- 返回最近物流单及阶段进度。

## 与后续阶段衔接
- Step 03 完成后，货物状态为“清关完成待入仓”。
- Step 04（海外仓与入仓）读取清关完成的物流单并执行入仓。
- 本阶段不直接做仓库上架，只负责跨境运输与清关闭环。

## 本阶段暂不包含
- 多市场线路（SG/TH 等）
- 异常事件（查验延误、港口拥堵、政策变动）
- 人工逐节点推进按钮（先用时间自动推进）

## 验收标准
- 能从采购订单中选择待发运单。
- 能选择货代线路与清关策略，并看到实时费用/时效预估。
- 确认发运后资金正确扣减，物流单成功落库。
- 物流状态可随时间自动推进至“清关完成”。
- 完成后可被 Step 04 正常读取并进入入仓流程。

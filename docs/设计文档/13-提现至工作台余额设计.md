# 13-提现至工作台余额设计

## 1. 目标
在 Shopee 财务模块补齐“提现”能力，实现从 `我的余额（RM）` 提现并转入工作台经营资金（RMB），形成跨模块资金闭环。

本期重点：**提现申请 + 汇率换算 + 工作台余额联动**。

---

## 2. 设计范围（V1）

### 2.1 本期包含
- `我的余额` 点击“提现”后可输入金额并提交。
- 提现金额从 Shopee 钱包余额扣减（记财务流水 `withdrawal`）。
- 提现成功后按固定汇率换算并转入工作台可用资金。
- 工作台“总资金/采购剩余”口径纳入提现转入金额。
- 全链路数据落库，刷新后不丢失。

### 2.2 本期不包含
- 提现审核、手续费、风控拦截、失败回退补偿工单。
- 分批到账、T+N 到账（V1 视为实时到账）。
- 动态汇率、历史汇率回溯（V1 固定汇率）。

---

## 3. 汇率与金额规则

### 3.1 固定汇率
- 本期固定汇率：`1 RM = 1.74 RMB`。
- 配置键建议：`RM_TO_RMB_RATE=1.74`（后端读取，默认 1.74）。

### 3.2 金额精度
- Shopee 钱包（RM）保留 2 位小数。
- 转入工作台（RMB）按 `提现RM * 1.74` 计算后，四舍五入到 2 位小数。

### 3.3 校验规则
- 提现金额必须 `> 0`。
- 提现金额必须 `<= wallet_balance`。
- 必须存在默认银行卡（无默认卡不允许提现）。

---

## 4. 业务流程
1. 玩家进入 `Shopee -> 我的余额`，点击“提现”。
2. 输入提现金额（RM）并提交。
3. 后端校验：对局归属、默认银行卡、余额是否充足。
4. 同一事务内完成：
   - 写入 Shopee 财务流水（`withdrawal`, `direction=out`）；
   - 写入工作台资金调账记录（`source=shopee_withdrawal`, `direction=in`）；
   - 返回提现结果（含 `withdraw_rm`、`credited_rmb`、`rate`）。
5. 前端刷新余额页；返回工作台后“总资金/采购剩余”自动体现新增资金。

---

## 5. 数据模型设计

### 5.1 复用表：`shopee_finance_ledger_entries`
- 用途：记录提现支出流水。
- 关键取值：
  - `entry_type = withdrawal`
  - `direction = out`
  - `amount = 提现金额(RM)`
  - `status = completed`
  - `remark = 提现至工作台（汇率 1:1.74）`

### 5.2 新增表：`game_run_cash_adjustments`（建议）
- 表用途：记录工作台资金调账（含 Shopee 提现转入）。
- 字段建议：
  - `id`
  - `run_id`：对局 ID
  - `user_id`：用户 ID
  - `source`：来源（`shopee_withdrawal`/`manual_adjustment`...）
  - `direction`：`in` / `out`
  - `amount`：金额（RMB）
  - `remark`：备注
  - `related_ledger_id`：关联 Shopee 流水 ID（可空）
  - `created_at`

### 5.3 索引与注释
- 索引建议：`run_id`、`user_id`、`source`、`created_at`。
- 若新增表/字段，必须补齐 table comment 与 column comment。

---

## 6. 接口设计

### 6.1 新增：提交提现
- `POST /shopee/runs/{run_id}/finance/withdraw`
- 入参：
  - `amount`（RM）
- 出参建议：
  - `wallet_balance`（提现后 RM 余额）
  - `withdraw_rm`
  - `credited_rmb`
  - `exchange_rate`（1.74）
  - `ledger_id`
  - `cash_adjustment_id`

### 6.2 口径调整：工作台资金汇总
- 现有：`GET /game/runs/{run_id}/procurement/cart-summary`
- 调整：
  - `total_cash = initial_cash + cash_adjust_in - cash_adjust_out`
  - `remaining_cash = total_cash - procurement_spent - logistics_spent - warehouse_spent`

---

## 7. 前端交互设计

### 7.1 我的余额页
- 点击“提现”打开弹窗。
- 弹窗字段：
  - 默认银行卡摘要（只读）
  - 可提现余额（RM）
  - 提现金额输入（RM）
  - 预计到账（RMB，按 1.74 实时计算）
- 提交成功：
  - Toast 提示“提现成功，已转入工作台资金”；
  - 刷新余额总览与流水列表。

### 7.2 工作台联动
- 工作台继续调用 `procurement/cart-summary`，无需新增页面接口。
- 右侧“阶段总览数据”中的“总资金/采购剩余”自动显示提现转入后金额。

---

## 8. 验收标准
- 点击提现可成功提交，余额页产生一条 `withdrawal` 流水。
- Shopee 钱包余额按提现金额正确扣减。
- 工作台总资金与采购剩余按 `1 RM = 1.74 RMB` 正确增加。
- 刷新页面后金额不丢失，口径一致。
- 无默认银行卡或余额不足时，后端正确拦截并返回可读错误。

# CBEC SIM 阶段设计文档（第五步）

## 阶段进度
- 第一步（已完成）：注册/登录
- 第二步（已完成）：开局准备
- 第三步（已完成）：选品与采购
- 第四步（已完成）：国际物流与清关
- 第五步（当前）：海外仓选择与入仓

## 第五步目标
- 将 Step03 已清关完成的货物转入海外仓。
- 玩家选择仓型方案与仓位距离，形成后续店铺运营的履约能力参数。
- 通过“地图风格预览”直观体现距离对成本与时效的影响。

## 本阶段定位
- 本阶段是“履约落地”环节，承接物流清关，衔接 Step05 店铺运营。
- 当前市场先支持 `MY`（马来西亚）。
- 重点是“选仓策略 + 入仓确认 + 成本时效影响”。

## 页面交互方案（地图风格预览）

### 1) 页面布局
- 左侧：仓型方案卡（A/B/C）
- 中间：马来地图预览区域（仿高德交互风格）
- 右侧：费用时效面板 + 入仓确认

### 2) 地图区（核心）
- 地图展示：马来主要履约区域（重点显示吉隆坡）
- 点位类型：
  - 近吉隆坡仓位点（近）
  - 远吉隆坡仓位点（远）
- 视觉元素：
  - 仓点 Marker（可点击）
  - 覆盖圈层（配送时效圈：快/中/慢）
  - 线路提示（港口/清关点到仓）
- 交互方式：
  - 点击 Marker 选中仓位
  - 鼠标悬浮显示仓位信息气泡
  - 仓型切换后，地图点位样式与指标实时刷新

### 3) 左侧仓型方案（3选1）
- 方案 A：Shopee 官方仓
  - 成本：高
  - 稳定性：高
  - 错漏率：低
- 方案 B：第三方仓
  - 成本：低
  - 稳定性：中
  - 错漏率：中高
- 方案 C：自建仓
  - 初始投入：最高
  - 稳定性：最高
  - 长期单件成本：低

### 4) 右侧决策面板
- 当前选择：仓型 + 仓位（近/远）
- 费用预估：
  - 一次性费用（如自建仓投入）
  - 入仓处理费
  - 仓租周期费用
  - 预计总仓储费用
- 履约能力：
  - 平均发货时效
  - 预计配送时效
  - 错漏发率
  - 可承载单量
- 按钮：`确认入仓并生效`

## 规则设计（V1）

### 1) 选择规则
- 必须先有 `清关完成` 的物流单才能入仓。
- 必须同时选择：
  - 一个仓型（A/B/C）
  - 一个仓位（近/远）
- 未选择完整时不可确认。

### 2) 成本规则
- 本次扣费 = 入仓处理费 + 仓租预扣 +（若首次自建仓则加一次性建设费）
- 扣款时点：点击确认入仓时立即扣减。
- 资金不足时：禁止确认并提示缺口。

### 3) 时效规则
- 近吉隆坡：配送时效优、成本高。
- 远吉隆坡：配送时效略慢、成本低。
- 仓型与距离共同决定 Step05 发货/配送表现。

### 4) 质量规则
- 官方仓：错漏率最低
- 第三方仓：错漏率较高
- 自建仓：错漏率最低（可设为长期优势）

## 与 Step05 的参数衔接
Step04 生效后，Step05 运营读取以下参数：
- `warehouse_mode`：official / third_party / self_built
- `warehouse_location`：near_kl / far_kl
- `delivery_eta_score`：时效评分
- `fulfillment_accuracy`：履约准确率
- `warehouse_cost_per_order`：单单仓储成本

这些参数直接影响：
- 店铺发货表现
- 买家评分与退款概率
- 利润率

## 接口建议（FastAPI）
- `GET /game/runs/{run_id}/warehouse/inbound-candidates`
  - 获取可入仓的清关完成物流单
- `GET /game/runs/{run_id}/warehouse/options`
  - 获取仓型与仓位参数配置
- `POST /game/runs/{run_id}/warehouse/strategy`
  - 提交并生效仓策略（含扣费）
- `POST /game/runs/{run_id}/warehouse/inbound`
  - 对指定物流单执行入仓
- `GET /game/runs/{run_id}/warehouse/summary`
  - 返回当前仓策略、费用、入仓状态、库存汇总

## 地图实现建议（前端）

### V1（推荐）
- 不接真实高德 API，先做“地图风格组件”
- 使用静态底图 + 交互点位 + 覆盖圈层
- 优点：稳定、可控、开发快、演示效果足够

### V2（后续）
- 可选接入真实地图 SDK（高德/Mapbox）
- 增加拖拽缩放、路径、更多城市点位

## 验收标准
- 玩家可通过地图点位完成仓位选择（近/远）。
- 仓型 + 仓位选择后，右侧费用/时效面板实时变化。
- 确认后成功扣费并生成入仓记录。
- Step05 能读取该仓策略参数并用于运营计算。
- 重新进入 Step04 时可恢复已选策略与入仓状态（数据库持久化）。

# 09-履约发货与物流时效结算设计

## 1. 目标
在“买家池已产单”基础上，补齐卖家履约与结算主链路，贴近 Shopee 官方卖家中心体验：
- 卖家在“待出货”执行安排发货。
- 生成快递单号（tracking_no）与面单号（waybill_no）。
- 订单进入物流轨迹并按节点推进到签收。
- 基于仓点到买家地理距离计算预计时效（ETA）。
- 按平台扣费规则完成佣金/手续费/净入账结算。

本期优先实现单人单店 MVP，不做复杂售后仲裁与多承运商对账。

## 2. 设计范围（V1）
### 2.1 本期包含
- To Ship -> Shipping -> Completed 状态机。
- 安排发货动作（生成 tracking_no / waybill_no）。
- 距离驱动 ETA 计算（仓点到买家城市）。
- Step03（国际物流清关）线路选择与 Step05（Shopee 履约）时效接通。
- 物流时间线（已创建运单、已揽件、运输中、派送中、已签收）。
- 结算试算与入账（佣金、支付手续费、运费成本/补贴、净入账）。

### 2.2 本期不包含
- 退款退货完整闭环（仅预留状态位）。
- 多承运商复杂路由优化。
- 广告/活动补贴分摊到单。
- 多币种跨国结算与税务拆分。

## 3. 核心流程（贴近 Shopee）
1. Step03 完成后，系统已落库线路参数（`logistics_shipments.forwarder_key/forwarder_label/transport_days`）。
2. 订单生成后进入 `toship/processing`，显示在“待出货”。
3. 卖家点击“安排发货”：
   - 选择物流渠道（仅限商品上架时启用的渠道）。
   - 读取当前对局最近一票 Step03 运单线路（经济线/标准线/快速线）作为“平台线路基准时效”。
   - 基于“海外仓坐标 -> 买家坐标”距离，计算该订单的游戏日运输时长（n 天）。
   - 生成 `tracking_no`、`waybill_no`。
   - 计算 `ship_by_at`、`eta_start_at`、`eta_end_at`。
4. 系统写入物流轨迹初始节点：
   - `label_created`（已创建运单）
5. 定时任务推进物流节点：
   - `picked_up` -> `in_transit` -> `out_for_delivery` -> `delivered`
6. 订单状态随节点推进：
   - `toship` -> `shipping` -> `completed`
7. 在 `delivered` 时触发结算入账：
   - 写入结算明细与净入账。

## 4. Step03 -> Step05 时效接通设计（重点）
### 4.1 数据来源
- 线路来源（Step03）：
  - 表：`logistics_shipments`
  - 字段：`forwarder_key`、`forwarder_label`、`transport_days`、`customs_days`
  - 取值策略：按 `run_id + user_id` 取最近一条（`created_at desc`）作为当前店铺履约线路配置。
- 海外仓来源（Step04）：
  - 表：`warehouse_strategies + warehouse_landmarks`
  - 目标：拿到仓点经纬度（`warehouse_lat/lng`）。
- 买家来源（买家池）：
  - 表：`sim_buyer_profiles`
  - 字段：`lat/lng/city_code`

### 4.2 线路时效边界（游戏日）
- 经济线（`economy`）：`8~18` 天
- 标准线（`standard`）：`5~12` 天
- 快速线（`express`）：`3~8` 天

> 你当前业务口径“快速线最多 8 天”在这里被硬约束；后续只改配置，不改算法。

### 4.3 距离到时效映射（同线路内）
- 先算仓到买家距离 `distance_km`（Haversine）。
- 距离分档（可配置）：
  - `0~80km`：近距离（取线路下界附近）
  - `80~300km`：中近距离
  - `300~800km`：中远距离
  - `>800km`：远距离（取线路上界附近）
- 输出：`transit_days_actual`（整数游戏天）。
  - 示例（快速线）：近距约 `3~4` 天，中距约 `5~6` 天，远距约 `7~8` 天。

### 4.4 发货时落库字段（建议扩展）
- `shopee_orders.delivery_line_key`：线路 key（economy/standard/express）
- `shopee_orders.delivery_line_label`：线路中文名
- `shopee_orders.transit_days_actual`：该订单实际运输天数（游戏天）
- `shopee_orders.promised_days_min` / `promised_days_max`：该线路承诺区间（用于前端展示和追责）

> V1 若暂不加字段，也至少在物流事件描述中记录 `line + transit_days_actual`，否则后续难排查时效异常。

### 4.5 ETA 计算（以游戏时间为准）
- `eta_start_at = shipped_at + transit_days_actual(天)`
- `eta_end_at = shipped_at + min(transit_days_actual + 1, promised_days_max)(天)`
- 所有加法基于游戏时钟（run tick），不是本机实时时钟。

### 4.6 物流节点推进与签收触发
- 从 `label_created` 到 `delivered` 不再按“固定每小时一节点”。
- 改为“按 `transit_days_actual` 占比推进”：
  - 前段：`picked_up`
  - 中段：`in_transit`
  - 末段：`out_for_delivery`
  - 到达 `shipped_at + transit_days_actual` 时才允许进入 `delivered`
- 这样可以保证不会出现“刚发货很快就已完成”的不真实现象。

## 5. 时效计算规则（距离驱动）
### 5.1 输入
- 仓点坐标：来自前置“海外仓选择”已落库地标（warehouse_landmarks / strategy）。
- 买家坐标：由买家城市映射（可先用城市中心点字典）。
- 物流渠道：标准快递/标准大件/快捷快递。

### 5.2 距离计算
- 使用球面距离（Haversine）得到 `distance_km`。

### 5.3 ETA 档位（MVP）
- `0~80km`：1~2天
- `80~300km`：2~3天
- `300~800km`：3~5天
- `>800km`：5~7天

### 5.4 渠道修正
- 快捷快递：`-0.5 ~ -1` 天
- 标准快递：`0` 天
- 标准大件：`+1` 天

最终：
- `eta_start_at = ship_out_at + adjusted_days_min`
- `eta_end_at = ship_out_at + adjusted_days_max`

## 6. 费用与结算规则（MVP）
定义：
- `buyer_payment`：买家实付
- `platform_commission`：平台佣金（按类目费率）
- `payment_fee`：支付手续费（按固定比例）
- `shipping_cost`：物流成本（按距离+渠道估算）
- `shipping_subsidy`：平台运费补贴（可先给固定比例/档位）
- `net_income = buyer_payment - platform_commission - payment_fee - shipping_cost + shipping_subsidy`

建议默认参数（可配置）：
- 佣金费率：`6%`
- 支付手续费：`2%`
- 物流基准成本：`base + distance_km * per_km`
- 补贴：`0~20%`（按渠道和活动策略）

## 8. 接口设计
### 8.1 安排发货
- `POST /shopee/runs/{run_id}/orders/{order_id}/ship`
- 入参：`shipping_channel`（可选，默认按订单渠道）
- 出参：
  - `order_id`
  - `tracking_no`
  - `waybill_no`
  - `delivery_line_key`
  - `delivery_line_label`
  - `transit_days_actual`
  - `eta_start_at`
  - `eta_end_at`
  - `process_status`
  - `type_bucket`

### 8.2 物流详情
- `GET /shopee/runs/{run_id}/orders/{order_id}/logistics`
- 出参：
  - 订单基础信息
  - 轨迹节点列表（按时间倒序）

### 8.3 结算详情
- `GET /shopee/runs/{run_id}/orders/{order_id}/settlement`
- 出参：
  - 买家实付
  - 佣金、支付手续费、运费、补贴
  - 净入账
  - 结算状态与结算时间

### 8.4 批量发货（可选）
- `POST /shopee/runs/{run_id}/orders/batch-ship`
- 入参：`order_ids[]`
- 出参：成功/失败清单

## 9. 前端交互（My Orders）
### 9.1 待出货
- 操作列三项：
  - 查看物流详情
  - 查看详情
  - 打印面单
- 新增主操作按钮：`安排发货`

### 9.2 运输中
- 显示物流渠道、tracking_no、当前节点、预计送达区间。

### 9.3 已完成
- 显示签收时间与净入账摘要，可点开结算明细。

## 10. 状态机约束
- 仅 `toship` 可执行 `ship`。
- 已 `shipping/completed/cancelled` 禁止重复安排发货。
- `delivered` 仅可从 `out_for_delivery` 进入。
- `delivered_at` 必须满足 `>= shipped_at + transit_days_actual(游戏天)`。
- 结算仅在 `delivered` 后可 `settled`。

## 11. 验收标准
- 订单可从待出货成功安排发货并生成单号。
- 物流轨迹能按节点推进并在详情页可见。
- ETA 与仓点-买家距离及渠道变化有显著差异。
- 同一线路下，近距离订单早于远距离订单签收；快速线订单整体快于标准线/经济线。
- 快速线订单时效上限不超过 8 游戏天。
- 签收后订单自动完成并生成结算记录。
- 结算明细可核对：`净入账 = 实付 - 佣金 - 手续费 - 运费 + 补贴`。

## 12. 实施顺序（建议）
1. 扩展订单字段 + 新增物流事件表 + 新增结算表。
2. 接入 Step03 线路读取（`logistics_shipments` 最新记录）与仓点坐标读取。
3. 完成 `ship` 接口并写入 `delivery_line + transit_days_actual + ETA`。
4. 完成物流时间线查询接口与前端详情展示。
5. 接入“按运输时长到达 delivered”的推进逻辑。
6. 完成结算计算与结算详情展示。
